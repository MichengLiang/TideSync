[#tidesync-runtime-path.domain]
== TideSync 运行路径

TideSync 当前运行路径由 Python 包依赖、agent 入口、环境变量、Vision Agents 插件导入和 GetStream edge 共同组成。这个路径决定 Qwen adapter 修改是否真的进入运行系统。

[#tidesync-python-dependency.domain]
=== Python 依赖事实

TideSync 当前 `pyproject.toml` 在 `[project].dependencies` 中声明 `vision-agents[qwen,getstream]>=0.6.4,<0.7`。这个依赖范围允许解析上游 Vision Agents v0.6.x 包。它没有指向 TideSync 仓库内的派生源码。

该事实产生一个验收约束：PR 中即使新增 adapter 源码，如果 `pyproject.toml`、workspace、editable install 或等价依赖解析没有让运行时加载该源码，适配工作仍未进入 TideSync 系统。

[#tidesync-agent-entry.domain]
=== Agent 入口事实

TideSync 当前 `src/tidesync/agent.py` 从 `vision_agents.plugins` 导入 `getstream` 和 `qwen`。`create_agent()` 调用 `qwen.Realtime(...)` 构造实时模型对象，并用 `Agent(edge=getstream.Edge(), llm=llm, ...)` 进入 Vision Agents realtime flow。

该入口说明 TideSync 并不直接管理 Qwen WebSocket 事件。Qwen WebSocket 输入、输出、响应 lifecycle、打断、工具、搜索和错误处理都发生在 `qwen.Realtime` provider 层。TideSync 外层只能传入配置参数，不能补齐 provider 内部状态机。

[#tidesync-settings-surface.domain]
=== 配置表面事实

`RealtimeSettings` 当前只包含 `model`、`base_url`、`voice` 和 `fps` 四个字段。默认模型是 `qwen3.5-omni-flash-realtime`，默认 WebSocket 地址是 `wss://dashscope.aliyuncs.com/api-ws/v1/realtime`，默认音色是 `Tina`，默认 FPS 是 1。

`create_agent()` 将这些字段传给 `qwen.Realtime`，并固定 `include_video=True`。TideSync 当前没有配置字段表达 `turn_detection.type`、`semantic_vad`、Manual 模式、`enable_search`、`search_options`、tools、audio format、input transcription model、usage 采集、错误策略或 tool/search 互斥规则。

该事实产生一个边界：TideSync 当前能够选择模型和少量输出参数，但不能让 Qwen3.5 Omni Realtime 的完整 WebSocket contract 成为一等配置面。09 号书后续的配置契约必须指定哪些字段由 adapter 内部合同固定，哪些字段通过 TideSync 配置暴露，哪些字段作为扩展配置保留。

[#runtime-path-evidence.contract]
=== 运行路径证据要求

运行路径必须被测试证明。符合实现需要提供一条测试或命令输出，显示 `vision_agents.plugins.qwen` 或等价 provider 模块路径位于 TideSync 仓库内受控源码目录，而不是 site-packages 的上游包路径。

运行路径证据必须与 PR diff 对应。若测试通过时依赖开发者本机临时 `PYTHONPATH`、未提交 editable clone 或 external 目录，证据无效。
