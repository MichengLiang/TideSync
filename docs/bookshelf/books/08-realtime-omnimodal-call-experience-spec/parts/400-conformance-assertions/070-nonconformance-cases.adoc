[#nonconformance-cases.nonconformance]
== 不符合案例

不符合案例列出常见误判。它们帮助评审者识别“看起来像”但不满足本书对象的实现。

[#photo-qa-is-not-realtime-vision.nonconformance]
=== 拍照问答不等于实时视觉通话

一个实现如果只允许用户拍照上传，再等待 AI 对单张图片回答，就不符合实时全模态视频通话体验。它可以是图片问答系统，但缺少当前视觉现场、持续会话、视觉变化和打断动态。

拍照问答可以作为辅助能力进入产品。它不能替代摄像头或等价视觉输入的当前现场能力。

[#voice-assistant-is-not-video-call.nonconformance]
=== 语音助手不等于视频通话体验

一个实现如果只能听用户说话并用语音回答，但不能使用当前视觉现场，就不符合实时全模态视频通话体验。

语音助手可以成为摄像头关闭后的受限模式。它不能在核心符合声明中冒充全模态体验。

[#avatar-is-not-vision-understanding.nonconformance]
=== 数字人形象不等于视觉理解

一个实现如果展示会说话的数字人形象，但 AI 不能使用用户摄像头或视觉现场，就不符合本书对象。

数字人形象是表达层扩展。它不证明系统能看见用户，也不证明系统能处理视觉指代或打断。

[#uninterruptible-output-nonconformance.nonconformance]
=== 不可打断输出不符合实时通话

一个实现如果 AI 回答期间用户不能通过新输入改变当前回答，就不符合核心体验。长回答必须播完、用户只能等待、插话只在回答结束后处理，都是不可打断输出。

不可打断输出可以出现在离线生成、播客式回答或批处理问答中。它不能构成实时视频通话体验。

[#hidden-capture-nonconformance.nonconformance]
=== 隐藏采集不符合体验契约

一个实现如果采集麦克风、摄像头或屏幕共享却没有用户可见状态，就不符合本书对象。高敏输入需要持续可观察边界。

隐藏采集即使技术上有效，也破坏用户控制权。用户不知道自己正在被听见或看见时，无法参与可信的实时通话。

[#model-capability-list-nonconformance.nonconformance]
=== 模型能力列表不等于产品符合

一个供应商如果只列出模型支持音频、视频、图像、文本、工具调用，不足以证明产品符合。模型能力需要通过用户表面的会话、状态、回应、打断和断言来体现。

符合性证据必须映射到本书断言。模型能力可以解释实现可能性，不能替代黑盒结果。

[#misleading-state-nonconformance.nonconformance]
=== 状态误导构成不符合

一个实现如果显示正在看但摄像头关闭，显示正在听但麦克风未授权，显示正在回应但没有可听输出，显示会话结束但继续采集输入，构成状态误导。

状态误导破坏公共投影。用户依靠状态决定是否展示现实、是否说话、是否停止采集。状态错误不是小 UI 问题，而是体验契约失败。
