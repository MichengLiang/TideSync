[#visual-reference-flow.flow]
== 视觉指代流

视觉指代流描述用户用语言指向当前视觉现场中的对象、位置、动作或状态时，系统如何完成理解。视觉指代是实时全模态视频通话体验的核心区别点之一。没有视觉指代，系统只能回答文字化的问题，无法承担“你看这个”的通话式交互。

视觉指代发生在语音文本本身不足以确定所指对象时。用户说“这个”“那里”“刚才那个”“屏幕左上角”“我手里这个”“这一步”时，语言只给出指向动作，所指对象需要从视觉现场和会话上下文中确定。

[#current-visual-reference.flow]
=== 当前视觉指代

当前视觉指代指向此刻视觉现场中的对象或位置。用户把摄像头对准药盒并说“这个怎么吃”，把屏幕对准错误弹窗并说“这是什么意思”，把手工步骤展示给 AI 并说“这一步对吗”，都属于当前视觉指代。

符合实现必须把当前视觉现场纳入解释。如果视觉输入可用且画面足以支持判断，AI 回应应与画面中的对象、文字、动作或位置相关。回应不需要复述全部视觉内容，但必须让用户能理解系统正在基于当前画面回答。

当前视觉指代不能被系统仅用语音文本替代。用户说“这个是什么”时，系统如果只回答“请描述一下这个东西”，说明它没有使用当前视觉现场。系统可以请求用户调整画面，但请求应当基于视觉受限事实，而不是把视觉指代退回文字问答。

[#historical-visual-reference.flow]
=== 历史视觉指代

历史视觉指代指向同一会话中已经出现过的视觉对象或状态。用户说“刚才那个红色按钮”“你前面看到的那张菜单”“刚刚我手里拿的那个”，需要系统在会话内视觉上下文中寻找对象。

历史视觉指代不要求无限记忆。符合实现应当承接合理的近期会话上下文，但可以在对象不再可靠、画面已切换太久、上下文不足或实现声明的记忆范围外时说明限制。

系统不得把历史视觉内容伪装成当前视觉现场。用户问“刚才那个是什么”时，系统可以基于历史上下文回答；用户问“现在这个是什么”时，系统必须使用当前画面。历史和当前的混淆会破坏视觉可信度。

[#deictic-language.flow]
=== 指示语言

指示语言包括“这个”“那个”“这里”“那里”“上面”“下面”“左边”“右边”“刚才”“现在”等表达。它们在日常对话中依赖共同注意。实时全模态视频通话体验要把用户和 AI 放进可共享的注意空间。

符合实现不需要完美解析所有指示语言，但必须对核心指示场景有可接受行为。用户指向画面中唯一显著对象时，系统应能回答。画面中有多个候选对象时，系统应当澄清。视觉现场不可用时，系统必须说明自己无法确定所指。

指示语言的失败不能被当成普通问答失败。它暴露的是全模态体验的关键能力。一个系统如果长期无法处理“这个”“那里”“刚才那个”，就无法核心符合。

[#visual-ambiguity.flow]
=== 视觉歧义

视觉歧义发生在画面中存在多个候选对象、对象不清楚、用户指向不明确、视角变化过快或语音表达不足时。歧义是正常现象。符合实现应当处理歧义，而不是假装确定。

处理歧义的合理方式包括：询问用户指的是哪一个，请用户靠近或稳定画面，请用户用手指示，描述自己看到的候选对象，或说明画面不足以判断。

不合理方式包括：随机选择一个对象作确定回答；把看不清说成看清；忽略视觉输入只给通用答案；要求用户完整输入文字描述而不承认视觉受限。

[#visual-change.flow]
=== 视觉变化

视觉现场会改变。用户移动摄像头、切换对象、翻页、靠近文字、转向屏幕、离开桌面或关闭摄像头时，系统对视觉现场的解释也应改变。

符合实现应当区分视觉变化带来的新当前现场。用户把摄像头从菜单移到饮料杯后，再问“这个怎么样”，系统应围绕饮料杯，而不是继续围绕菜单。用户关闭摄像头后，再问“你看这个”，系统应说明当前没有视觉输入。

视觉变化也影响历史上下文。系统可以记得刚才看过菜单，但不能把菜单当成当前画面。用户如果追问历史对象，系统需要使用历史指代语义；用户如果追问当前对象，系统需要使用当前视觉现场。

[#visual-reference-completion.flow]
=== 视觉指代完成

视觉指代完成时，系统已经把用户的指示语言、当前或历史视觉现场、会话上下文和用户意图放进同一解释中，并以自然语音给出相关回应或诚实限制。

完成可以有三种结果。第一，系统确定所指并回答。第二，系统发现歧义并请求澄清。第三，系统无法使用视觉现场并表达限制。这三种都可以符合体验。伪装确定、忽略视觉、沉默失败和使用错误视觉现场，不符合体验。

视觉指代流是符合性断言的核心来源。能够处理视觉指代，才说明系统不是把摄像头当装饰，而是在会话中使用视觉现场。
