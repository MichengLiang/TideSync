[#visual-evidence-reference-assets.concept]
== 视觉证据与引用资产

视觉证据是进入会话并可支持回答的视觉材料。引用资产是被明确捕获、编号、保存于当前会话范围内，并可被用户后续指称的视觉证据。快照、短视频片段、屏幕片段和特定工具返回的视觉材料，都可以成为引用资产。

引用资产的价值不只是降低成本。它把动态、易漂移的视觉现场转化为稳定、可回指、可审查的会话对象。用户围绕 `图 1` 或 `片段 1` 继续追问时，系统不必持续依赖当前摄像头画面，也不必猜测“刚才那个”到底指向哪一帧。

[#current-visual-evidence.contract]
=== 当前视觉证据

当前视觉证据来自摄像头、屏幕共享或等价视觉输入的当前现场。它服务xref:current-visual-reference[当前视觉指代, rel=depends-on]，支持用户说“这个”“现在这里”“你看我手里这个”时的回答。

当前视觉证据需要满足视觉新鲜度。用户移动摄像头、切换窗口、翻页或改变操作步骤后，旧证据可能不再支持当前问题。系统无法确认当前证据时，应表达限制。

当前视觉证据不自动成为引用资产。系统可以使用当前画面回答一轮问题，但除非实现明确捕获、编号或保存，否则用户后续说“图 1”或“那个片段”时，系统不能假装存在稳定资产。

[#snapshot-reference-asset.contract]
=== 快照引用资产

快照引用资产是会话内被捕获为单张图像的视觉证据。它应具有用户可理解的引用标识，例如 `图 1`、缩略图、时间戳或等价标记。

快照引用资产适合静态对象、静态文本、菜单、药盒、植物、设备面板、错误弹窗和文档局部。它让用户可以围绕同一视觉证据多轮追问，而不会被后续摄像头移动干扰。

快照引用资产需要记录边界。系统应能区分快照内可见内容、快照外不可见内容、快照捕获后的现实变化。用户问“图 1 左上角是什么”时，系统应基于快照；用户问“现在这个是什么”时，系统应基于当前视觉或说明当前视觉不可用。

[#clip-reference-asset.contract]
=== 片段引用资产

片段引用资产是会话内被捕获为有限时长视频、屏幕录制或等价动态材料的视觉证据。它应具有用户可理解的起止状态和引用标识，例如 `片段 1`。

片段引用资产适合过程性任务。用户可以录下设备故障灯闪烁、手工操作步骤、屏幕错误复现、菜单滚动、手势动作或短时间环境变化，然后围绕片段进行问答。

片段引用资产必须保留起止语义。系统只能基于片段时间范围内的内容作判断。片段结束后发生的现实变化，不属于该资产。系统需要新证据时，应请求用户继续录制、切换到实时视觉或捕获新片段。

[#screen-reference-asset.contract]
=== 屏幕引用资产

屏幕引用资产来自屏幕共享、窗口截图、屏幕片段或等价桌面视觉输入。它通常包含更高隐私风险，因为屏幕可能暴露账号、消息、文件、代码、业务数据或通知。

屏幕引用资产应明确来源和范围。用户需要知道系统捕获的是整个屏幕、某个窗口、某个标签页、某张截图还是某段屏幕录制。来源不清会导致用户误判隐私暴露和回答依据。

屏幕引用资产的保存和删除语义应比普通视觉资产更清楚。实现声明保存屏幕材料时，应说明保存范围、访问范围和删除路径。

[#reference-asset-lifecycle.contract]
=== 引用资产生命周期

引用资产生命周期包括捕获、编号、进入会话、被引用、失效、删除和会话结束后的处理。生命周期状态必须可观察到足以支撑用户动作。

捕获时，用户需要知道系统正在拍照、录制片段或捕获屏幕。编号时，用户需要知道该资产如何被后续引用。引用时，用户需要知道回答依据的是哪个资产。失效时，用户需要知道资产已过期、不可用、被删除或不再适合当前问题。

引用资产不自动跨会话存在。核心符合只要求同一会话内上下文连续。实现如果保存引用资产用于长期记忆、历史记录、训练、摘要或跨会话检索，必须进入xref:privacy-after-session[会话结束后的隐私, rel=depends-on]和xref:memory-boundary[长期记忆边界, rel=depends-on]。

[#reference-asset-privacy.contract]
=== 引用资产隐私

引用资产可能比实时流更容易被忽视，因为它们从“正在采集”转为“已经捕获”。已经捕获不等于隐私风险消失。图像、片段和屏幕材料仍可能包含个人、医疗、财务、身份、业务、位置或账号信息。

符合实现应让用户知道引用资产是否仅用于当前会话、是否保存到记录、是否上传云端、是否进入工具、是否可删除。用户删除资产后，系统不得继续把该资产作为当前可用证据，除非存在另一个被用户理解的保存范围。

引用资产隐私也约束成本优化。实现不能为了降低重复视觉处理成本而秘密保存图像或片段。缓存和保存一旦影响用户数据边界，就必须成为公共契约的一部分。

[#reference-asset-conformance.contract]
=== 引用资产符合性

引用资产符合性要求系统保持三种区分。

第一，当前视觉现场与引用资产不同。当前视觉回答必须依据当前视觉或说明不可用；引用资产回答必须依据被引用对象。

第二，引用资产与历史视觉上下文不同。历史视觉可能只是系统曾经看过；引用资产是被明确捕获并可被用户后续指称的证据。

第三，引用资产与长期记忆不同。引用资产可以在当前会话内支持多轮讨论；跨会话使用需要独立声明。

这些区分使成本控制不破坏语义清晰。快照和片段可以降低持续视觉成本，但它们不能把已捕获证据伪装成当前现实，也不能把当前现实伪装成已编号资产。
