[#unit-economics-and-allocation.contract]
== 单位经济与成本分摊

单位经济把资源消耗关联到产品、场景、任务和业务结果。成本分摊把资源消耗归属到用户、租户、平台、模态、工具、模型和功能边界。两者共同回答：成本花在什么地方，产生了什么体验价值。

实时全模态视频通话体验不适合只用“每分钟成本”或“每次请求成本”衡量。用户获得价值的单位可能是一次成功视觉回答、一次完成的操作指导、一次工具增强结果、一个可复用引用资产、一次成功打断恢复，或一个完成的业务任务。

本章的单位经济语言参考 FinOps 对成本可见、分摊、业务价值和单位经济的处理方式。FinOps 坐标用于说明成本治理如何服务工程、财务和业务共同决策，不构成本书的符合性权威。参见 <<finops-framework>>、<<finops-unit-economics>>、<<finops-ai>>、<<aws-cost-optimization>> 与 <<google-cloud-cost-optimization>>。

[#cost-allocation-object.contract]
=== 成本分摊对象

成本分摊对象是成本归属的目标。符合实现可以按用户、租户、平台、设备、地区、会话、任务、模态、模型、工具、引用资产或业务流程分摊成本。

不同分摊对象服务不同判断。按会话分摊可以观察总体运营压力；按任务分摊可以判断业务价值；按模态分摊可以判断视觉或语音是否过度消耗；按工具分摊可以判断外部能力是否值得；按引用资产分摊可以判断快照和片段是否降低了持续实时成本。

成本分摊不应改变用户承诺。实现可以在后台把成本归属给某个租户或功能，但用户表面仍应保持状态、权限、错误和降级语义一致。

[#resource-unit-metrics.contract]
=== 资源单位指标

资源单位指标描述底层资源效率。它们适合工程和运营使用，但不能单独证明用户价值。

资源单位指标可以包括每分钟实时连接成本、每分钟音频输入成本、每张快照处理成本、每秒片段处理成本、每次语音输出成本、每次工具调用成本、每轮模型推理成本、每 MB 传输成本、每 MB 存储成本和每次上下文摘要成本。

资源单位指标能够定位浪费。例如某平台每张快照处理成本异常高，可能说明图像压缩、模型路由或缓存策略存在问题。资源单位指标不能单独决定产品策略，因为低成本资源路径可能没有完成用户任务。

[#business-unit-metrics.contract]
=== 业务单位指标

业务单位指标描述资源消耗与用户任务结果之间的关系。它们更适合产品、运营、财务和供应商评审。

业务单位指标可以包括每个成功会话成本、每个成功视觉回答成本、每个完成任务成本、每个工具增强有效回答成本、每个被用户继续追问的引用资产成本、每个成功打断恢复成本、每个受限后恢复成功成本和每个业务动作成功成本。

业务单位指标帮助识别策略价值。连续实时模式成本高，但如果它显著提高操作指导成功率，可能合理。快照模式成本低，但如果用户需要频繁重拍且任务失败率高，可能只是表面省钱。

[#unit-economics-formulas.contract]
=== 单位经济公式

单位经济可以使用简单公式表达成本与结果的关系。公式服务比较，不服务装饰。

[source,text]
----
CostPerSession = TotalSessionCost / SessionCount
CostPerSuccessfulTask = TotalCost / SuccessfulTaskCount
CostPerVisualAnswer = VisualRelatedCost / ValidVisualAnswerCount
CostPerReferenceAsset = ReferenceAssetCost / ReferencedAssetUseCount
CostPerToolGroundedAnswer = ToolCost / ValidToolGroundedAnswerCount
CostPerInterruptionRecovery = InterruptionHandlingCost / SuccessfulInterruptionCount
----

公式中的分母必须有可观察定义。`ValidVisualAnswerCount` 不能只表示模型生成过视觉回答；它应表示回答在声明范围内基于正确视觉证据、没有状态误导，并通过相应断言。`SuccessfulTaskCount` 也应基于产品定义的任务完成条件。

单位经济公式不替代符合性断言。一个路径即使单位成本低，只要它失败于核心断言，就不能作为核心符合策略。

[#allocation-and-conformance.contract]
=== 成本分摊与符合性声明

成本分摊结果可能影响xref:conformance-statement[符合性声明, rel=depends-on]。实现改变采样策略、模型服务、工具限制、会话时长、保存策略或平台范围时，可能改变适用断言和证据。

符合性声明不需要公开全部商业成本，但应说明影响体验范围的成本边界。例如某账号层级限制连续实时视觉时长，某地区不支持低延迟模型服务，某平台不支持屏幕共享，某工具因配额限制仅在确认后调用，这些都属于范围或条件信息。

成本分摊也应支持异常发现。某类任务成本异常升高，可能说明用户被迫使用错误模式；某类工具成本高但有效回答低，可能说明工具触发条件不合格；某类快照资产很少被引用，可能说明控制面没有服务真实动作。

[#unit-economics-principle.contract]
=== 单位经济原则

单位经济原则是：成本必须与体验结果和业务结果关联，不能只与资源消耗关联。符合实现应同时观察资源单位指标和业务单位指标，并把成本优化指向高成本低价值路径，而不是平均压低所有能力。

这个原则保护高价值体验。实时全模态会话中的某些路径天然昂贵，例如高风险维修、远程教学、业务工具动作和高敏屏幕协助。成本治理要判断这些路径是否值得，而不是因为它们昂贵就自动降级。
