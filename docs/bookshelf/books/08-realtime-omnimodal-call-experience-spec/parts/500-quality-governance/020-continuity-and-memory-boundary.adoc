[#continuity-and-memory-boundary.boundary]
== 连续性与记忆边界

连续性让用户感到自己在同一会话中与同一个 AI 交流。记忆边界防止系统把没有资格使用的信息伪装成上下文。

本书核心符合要求会话内上下文连续，不要求跨会话长期记忆。长期记忆是条件能力或扩展能力，需要单独声明。

[#session-continuity.contract]
=== 会话内连续性

会话内连续性包括用户刚说过的话、刚展示的视觉现场、AI 刚给出的回答、用户刚作出的纠正和工具刚返回的结果。它们共同构成当前会话解释范围。

符合实现应当承接合理的近期上下文。用户追问“为什么”“那这个呢”“你刚才说的第二点”，系统应能把这些表达放回会话中解释，或说明无法确定所指。

连续性不是保存一切。画面快速变化、权限关闭、会话结束、时间过长或实现范围限制都可能使上下文失效。失效需要诚实表达。

[#memory-boundary.contract]
=== 长期记忆边界

长期记忆指跨会话保存和使用用户信息、视觉内容、偏好、历史问题或业务数据。长期记忆不属于核心符合。

实现声明长期记忆时，必须说明保存什么、如何使用、何时生效、用户如何查看、修改、撤回或删除。没有这些公共语义，长期记忆会破坏隐私和信任。

实现未声明长期记忆时，不得暗示自己跨会话记得用户展示过的视觉内容或说过的敏感信息。它可以记住当前会话内信息，但不能越过会话边界。

[#context-honesty.contract]
=== 上下文诚实

上下文诚实要求系统承认自己不知道、没看到、没听清或无法承接的事实。诚实表达比伪确定回答更符合体验。

系统可以说“我刚才没有看到那一段”“摄像头关闭时我无法知道画面变化”“我不确定你说的是哪一个”。这些表达降低用户不确定性，并保护用户免受错误引导。

上下文诚实也要求系统不把工具结果、历史视觉、模型常识和当前视觉混为一谈。用户需要知道回答依据的大致来源。
