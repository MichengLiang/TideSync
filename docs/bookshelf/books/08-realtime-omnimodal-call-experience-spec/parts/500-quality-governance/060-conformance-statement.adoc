[#conformance-statement.contract]
== 符合性声明

符合性声明是实现者对本书要求的公开回答。它把实现范围、通过断言、条件限制、扩展能力、偏离项和证据材料放在同一份声明中。

符合性声明不是营销页。它必须可审查。评审者应能从声明追踪到本书标题、断言和证据。

[#statement-scope.contract]
=== 声明范围

声明范围说明实现在哪些平台、设备、地区、账号、网络条件、输入模态和业务场景中声称符合。范围应明确到足以选择对应断言。

范围可以有限。有限范围不是问题；未声明范围才是问题。一个移动端实现可以声明移动端核心符合；一个只支持摄像头不支持屏幕共享的实现可以声明核心符合但不声明屏幕共享扩展符合。

[#statement-coverage.contract]
=== 断言覆盖

符合性声明应列出适用断言及其结果。结果可以是通过、不适用、偏离或失败。每个不适用和偏离都需要理由。

断言结果应链接到证据。证据可以是测试记录、人工验收、屏幕录制、用户操作脚本或审查材料。证据必须能让评审者观察用户动作和系统结果。

[#statement-deviation.contract]
=== 偏离项

偏离项说明实现未满足某项应当要求、条件要求或扩展要求的情况。偏离项应说明影响用户、影响断言和替代行为。

偏离项不能取消核心必须项。缺少实时语音、当前视觉现场、自然语音回应、打断或公共投影时，实现不能用偏离项获得核心符合。

[#statement-evidence.contract]
=== 证据材料

证据材料应围绕黑盒行为组织。优先证据是可重复操作脚本和可观察结果记录。内部架构图、模型能力列表和宣传视频只能作为辅助材料。

证据材料需要版本、环境和范围。没有环境范围的证据不能说明条件符合。没有版本的证据无法支持持续审查。

[#statement-maintenance.contract]
=== 声明维护

符合性声明需要随实现变化维护。新增平台、新增工具、修改权限流、改变模型服务、改变采样策略、改变隐私保存，都可能影响符合性。

声明维护不是版本流水账。它只记录影响本书要求、断言、范围或证据的变化。无关内部重构不需要进入符合性声明。
