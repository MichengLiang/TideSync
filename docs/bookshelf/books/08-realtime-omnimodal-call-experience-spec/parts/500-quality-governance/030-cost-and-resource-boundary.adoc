[#cost-and-resource-boundary.boundary]
== 成本与资源边界

成本与资源边界描述实现如何控制音频、视觉、模型、工具和存储消耗，同时不破坏核心体验承诺。成本控制是现实约束；秘密降级不是合格体验。

实时全模态会话消耗资源。持续音频、视觉输入、语音输出、工具调用、上下文保存和日志都可能产生费用或配额消耗。实现可以控制资源，但必须守住用户可依赖的行为。

[#visual-sampling-boundary.contract]
=== 视觉采样边界

视觉采样是实现如何把视觉现场纳入会话的内部策略。实现可以持续处理视频轨道，也可以按频率或事件采样图像帧。本书不规定采样方式。

采样策略不得破坏当前视觉承诺。用户切换画面并提问时，系统不能长期使用旧画面。系统因采样限制无法可靠使用新画面时，应表达限制或请求用户等待、稳定画面。

采样频率不应作为营销替代符合性。高帧率不自动证明理解，低帧率不自动不符合。符合性看用户问题是否被正确放入当前视觉现场。

[#session-duration-boundary.contract]
=== 会话时长边界

实现可以限制会话时长。限制需要公开，并在接近限制时投影给用户。会话突然中断且无说明，会破坏体验。

会话时长限制影响上下文连续性。会话结束后，核心上下文边界关闭。实现如果提供继续会话、摘要或长期记忆，需要说明这些能力的范围。

[#tool-cost-boundary.contract]
=== 工具成本边界

工具调用可能消耗费用、配额、外部服务资源或业务操作机会。实现可以限制工具频率、工具类型或高成本工具使用。

限制工具调用时，系统不得给出依赖未调用工具的确定回答。它可以说明当前未使用外部信息，或请求用户授权高成本操作。

工具成本边界应与用户权益相关。低层内部优化无需打扰用户；影响回答事实、业务动作、费用或等待时间时，需要公共投影。

[#resource-degradation-boundary.contract]
=== 资源降级边界

资源降级发生在系统为了成本、性能或故障处理降低能力。降级可以合理存在，例如降低视觉采样、关闭工具、减少记录、转入语音受限模式。

降级必须与符合性声明一致。核心能力降级时，系统应进入受限状态。扩展能力降级时，系统应说明扩展不可用。降级不能悄悄把全模态体验变成语音助手或图片问答。
