[#accessibility-and-environment.contract]
== 可访问性与环境适应

可访问性与环境适应描述系统如何在不同用户能力和环境条件下保持体验可理解。它们不改变核心对象，但影响真实消费级使用。

[#noisy-environment.contract]
=== 嘈杂环境

嘈杂环境会影响语音输入和输出。系统可能听不清用户，也可能让用户听不清 AI。符合实现应当在识别不可靠时表达限制，而不是伪装听懂。

字幕、转录、耳机提示、重新说话请求和降噪能力都可以帮助嘈杂环境。实现声明这些能力时，需要保持状态一致。

[#low-vision-and-hearing.contract]
=== 视觉与听觉差异

听障用户可能依赖字幕或文字记录。视障用户可能依赖语音状态和非视觉反馈。实现可以通过字幕、语音提示、键盘操作、屏幕阅读器支持和高对比状态增强可访问性。

可访问性增强属于重要质量能力。它不取消自然语音核心，但可以为不同用户提供等价理解路径。实现声明可访问性扩展时，应给出可观察行为。

[#mobile-and-desktop-environment.contract]
=== 移动端与桌面端环境

移动端和桌面端的权限、摄像头、麦克风、屏幕共享、后台运行和系统隐私指示不同。实现可以声明支持范围。

支持范围必须进入条件符合声明。只支持移动端的实现不需要通过桌面端断言；但它必须在移动端满足核心体验。支持桌面端屏幕共享的实现需要处理屏幕共享条件责任。

[#weak-network-environment.contract]
=== 弱网环境

弱网环境会影响语音连续性、视觉更新、模型响应和工具调用。系统可以降级，但降级必须可见。

弱网下合理行为包括提示网络不稳定、降低视觉更新、请求用户等待、转入语音受限或结束会话。不合理行为包括无状态沉默、旧画面回答新问题、继续显示能力正常。
