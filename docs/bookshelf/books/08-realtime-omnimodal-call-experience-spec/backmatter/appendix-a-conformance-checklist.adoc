[#appendix-conformance-checklist]
= 附录 A：符合性检查表

本检查表用于审查实现是否有资格声明核心符合。

[cols="1,3", options="header"]
|===
|检查项 |判定问题

|对象身份
|实现是否提供同一会话中的语音输入、当前视觉现场、自然语音回应和可打断动态语义。

|设备权限
|麦克风、摄像头和条件视觉输入的授权、拒绝、撤回和恢复是否具有可观察状态。

|当前视觉现场
|用户能否询问当前画面中的对象、文字、位置或动作；系统无法可靠判断时是否诚实表达。

|自然语音回应
|AI 是否以用户可听的自然语音承担主要回答；语音不可用时是否进入受限状态。

|打断
|用户能否在 AI 回答期间通过新输入停止或重定向当前回答。

|上下文连续
|系统是否承接同一会话中的相关语音、视觉和回应历史，并在缺失时说明边界。

|公共投影
|用户是否能知道系统正在听、正在看、正在处理、正在回应、已受限、出错或已结束。

|错误恢复
|权限、网络、设备、模型和工具错误是否被投影，并提供恢复或结束路径。

|隐私采集
|高敏输入采集是否有持续可观察状态，用户是否能停止采集。

|工具与外部动作
|搜索、位置、业务 API、设备控制或其他工具能力是否具有授权、结果、失败和成本投影。

|成本治理
|成本策略是否说明降低了哪类资源消耗，是否触碰核心符合门槛，是否在影响用户可依赖行为时投影模式、降级或限制。

|视觉引用资产
|快照、片段或屏幕材料是否具有明确编号、起止、引用、失效、删除和隐私边界；系统是否区分引用资产、当前视觉现场和历史视觉上下文。

|单位经济
|实现是否能把成本归因到会话、任务、模态、工具、引用资产或业务结果，而不是只提供总账单下降或模型调用次数减少。

|符合性证据
|实现是否提交可映射到本书断言和成本断言的黑盒证据，而不是只提交模型能力、演示视频、账单截图或内部架构图。
|===
